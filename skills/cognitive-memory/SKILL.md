# 認知記憶系統使用指南

## 你擁有的記憶工具

透過 MCP Server `cognitive-memory`，你可以使用以下工具：

### 記憶存取
- `save_memory` — 儲存重要資訊。設定 importance ≥ 0.7 的記憶會在下次對話自動載入。
- `recall_memory` — 用擴散激活搜尋記憶。輸入關鍵字，系統會找到直接和間接關聯的記憶。
- `forget_memory` — 刪除一條記憶。
- `list_memories` — 查看記憶庫概覽。

### 記憶整理
- `trigger_sleep` — 手動觸發記憶鞏固（模擬睡眠）。會執行重播、修剪、模式提取、整合四個階段。
- `sleep_status` — 查看上次鞏固時間和記憶網路統計。

## 記憶使用原則

1. **自然使用**：像一個記憶力好的人那樣使用記憶，不要說「根據我的記憶系統」
2. **主動儲存**：當使用者提到姓名、偏好、專案等重要資訊時，主動用 `save_memory` 記住
3. **適當查詢**：不確定時用 `recall_memory` 查詢，不要猜測
4. **情緒標記**：儲存記憶時注意情緒標記，正面/負面/緊急的內容給更高 importance
5. **標籤關聯**：給記憶加上有意義的 tags，這些 tags 會用於擴散激活找到關聯記憶

## 記憶分類

- `fact` — 事實（姓名、職業、技術棧）→ importance 0.8-0.95
- `preference` — 偏好（喜歡的東西、工作風格）→ importance 0.6-0.8
- `context` — 上下文（當前專案、目標）→ importance 0.7-0.85
- `episodic` — 情節（發生過的具體事件）→ importance 0.4-0.7

## 自動行為

- **SessionStart Hook** 會在每次啟動時檢查是否需要記憶鞏固
- 鞏固超過 24 小時未執行時會自動觸發
- 鞏固過程：重播加強 → 修剪弱連結 → 提取模式 → 整合新舊記憶
